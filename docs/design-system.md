*Visual and emotional meaning of interface components — so every future design stays consistent.*

## Button System

The app uses four button styles. These colours represent emotional intent rather than simple UI states.

### Green Button

Represents accepting an invitation from Tilly.

It means:

* Let's go
* Let's do it together
* I'm coming
* Yes
* Okay

This button should be used whenever the user chooses to participate in a shared ritual or act of self-care.

Examples:

* Let's Go
* Let's Walk
* Let's Eat
* Let's Get Water
* Let's Rest
* Let's Go Outside
* Let's Take Our Medicine

The green button represents moving towards life, care and companionship.

---

### Red Button

Represents postponing or declining without guilt.

It should never imply failure or punishment.

It means:

* Later
* Not right now
* Maybe later
* I'll stay here
* Not today

The app should never shame the user for choosing this option.

Tilly simply accepts the choice, settles down, waits quietly or checks in again later.

Examples:

* Later
* Maybe Later
* Not Right Now
* Stay Here

---

### Purple Button

Represents connection rather than action.

Purple is used for moments where the user simply wants to spend time with Tilly.

Examples include:

* Hello
* I Miss You
* Tell Me Something
* Sit With Me
* Cuddle
* Remember
* Good Morning
* Good Night

Purple buttons should feel emotionally warm and companionable rather than task-oriented.

---

### Grey-Blue Button

Grey-blue is not an emotional colour.

It is the universal interaction state used for pressed, hover or active feedback across all buttons.

It communicates physical interaction only and should not carry semantic meaning.

---

## Design Philosophy

Buttons should never feel like productivity software.

The interface should never feel like checking off tasks.

Instead, every interaction should feel like responding to a gentle invitation from Tilly to share an ordinary moment together.

Whenever possible, prefer language like:

* Let's Go
* Come With Me
* Let's Eat
* Let's Get Water
* Let's Rest

over language like:

* Complete
* Confirm
* Submit
* Done
* Finish

The app should always reinforce companionship over compliance.
